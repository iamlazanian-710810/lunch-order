# 辦公室午餐訂餐系統 — 複製建立指南

> 用 Claude 全程協助建立，約半天完成，完全免費部署。

---

## 系統功能總覽

| 頁面 | 誰用 | 功能 |
|------|------|------|
| 主頁 `/` | 所有人 | 看今日菜單圖 → 填餐點／價格／備註 → 送出；可刪除自己的訂單 |
| 後台 `/admin` | 管理員（需 PIN） | 上傳菜單圖、查看訂單、複製叫餐清單、本月每日明細 |
| 報表 `/report` | 管理員 | 月結每人費用、每日明細展開、匯出 CSV |

---

## 後台功能細節

### 今日設定
- 上傳今日菜單圖片（手機直接拍照）
- 員工在主頁可點圖放大查看
- 即時訂單：**依員工** 或 **依品項** 兩種視角切換
  - 依員工：每人每筆列出，可單筆刪除
  - 依品項：同餐點合併，備註不同則分開，顯示誰點了、幾份
- 一鍵複製叫餐清單（打電話用）

### 本月記錄
- 依日期分組展開，最新在上
- 每筆可單獨刪除
- 顯示員工、餐點、備註、金額

### 餐廳管理
- 新增／刪除餐廳與菜單（供參考用）

### 員工管理
- 新增／刪除員工名單

## 技術架構

| 項目 | 選擇 | 費用 |
|------|------|------|
| 前後端 | Next.js 16（App Router） | 免費 |
| 資料庫 | Supabase（PostgreSQL） | 免費方案 |
| 部署 | Vercel | 免費方案 |
| 身分識別 | 下拉選名字（無帳密） | — |
| 後台保護 | PIN 碼（環境變數設定） | — |

**容量說明：** 點餐文字資料幾乎不佔空間；菜單圖片每張壓縮後約 200–300 KB，上傳時自動清除 90 天前的舊圖，免費方案（500 MB）可用數年。

## 日常使用流程

```
每天早上（管理員）
  → /admin 輸入 PIN → 今日設定 → 上傳菜單照片

中午前（所有人）
  → 主頁 → 選自己名字 → 看菜單圖（可點放大）→ 填餐點/價格/備註 → 送出

訂餐截止後（管理員）
  → 今日設定 → 切換「依品項」→ 複製叫餐清單 → 打電話訂餐

月底（管理員）
  → /report → 選月份 → 對帳 → 匯出 CSV 收款
```

## 修改 PIN 碼

在 Vercel 後台修改環境變數 `NEXT_PUBLIC_ADMIN_PIN`，重新部署即生效。
